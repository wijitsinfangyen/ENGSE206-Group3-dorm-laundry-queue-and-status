# Final Package and Submission

> Week 16 ใช้โฟลเดอร์นี้เก็บไฟล์ export หรือเอกสารสรุปที่จะส่งตามที่อาจารย์กำหนด

## Final Deliverables Checklist

- [ ] `final-srs.pdf` หรือเอกสาร SRS ตามรูปแบบที่กำหนด
- [ ] `final-design-specification.pdf` หรือชุดเอกสารออกแบบ
- [ ] `final-presentation.pdf` / `.pptx`
- [ ] ลิงก์ prototype หรือไฟล์ prototype export
- [ ] `final-project-summary.md`
- [ ] หลักฐาน review/usability test
- [ ] Individual Reflection ของสมาชิกทุกคน
- [ ] Repository มีเอกสาร source ที่แก้ไขได้ครบ

## Recommended Final Folder Structure

```text
final/
├── README.md
├── final-project-summary.md
├── final-srs.pdf
├── final-design-specification.pdf
├── final-presentation.pdf
├── final-presentation.pptx
├── prototype-link.md
└── midterm-snapshot/          # ใช้เมื่ออาจารย์กำหนด
```

## Naming Convention

ใช้ `groupXX-projectname-artifact.ext` เช่น

- `group03-campus-space-booking-final-srs.pdf`
- `group03-campus-space-booking-final-design.pdf`
