# Software Design Artefacts

โฟลเดอร์นี้เป็นผลต่อเนื่องจาก SRS ที่ผ่านการตรวจสอบแล้ว ต้องอ้างอิง requirement ID และ rationale ทุกครั้งที่ตัดสินใจสำคัญ

1. `10-design-strategy.md` — แนวทาง/หลักการออกแบบและ trade-offs
2. `11-conceptual-architecture.md` — สถาปัตยกรรมระดับแนวคิด
3. `12-ux-ui-prototype.md` — user flow, wireframe/prototype, accessibility และ usability plan
4. `13-detailed-design.md` — data/API/interaction/detail design
5. `14-design-review.md` — review findings และ action plan
